# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.x.x   | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** open a public issue
2. Email the details to the repository owner
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### Response Timeline

- **Initial response:** Within 48 hours
- **Status update:** Within 7 days
- **Fix release:** Depends on severity

### What to Expect

- Acknowledgment of your report
- Regular updates on progress
- Credit in the release notes (unless you prefer anonymity)

## Security Best Practices

When using this MCP server:

1. **Never commit your Discord bot token** to version control
2. Use environment variables or secure config files
3. Regularly rotate your bot token
4. Use minimal required permissions for your bot
5. Keep dependencies updated

## Dependencies

This project uses automated dependency updates via Dependabot to ensure security patches are applied promptly.
